Salvando Alterações no Repositório Local

Quando alteramos o nosso repositório, é marcado pelo prórpio Git que há arquivos não rastreados, ou seja, que não estão salvos, como a imagem ao lado:

O novo arquivo, criado com o comando touch, fez com que houvesse alterações não salvas no versionamento do arquivo, sendo exibido no comando git status.
Para efetuar a adição desse novo arquivo, podemos utilizar o comando que o próprio Git cita, git add (arquivo).


Após a execução do git add, com a utilização do git status, para verificar a situação dos arquivos, podemos ver que houve uma mudança, que houve um novo arquivo adicionado para commit.


Com esse arquivo adicionado à área de preparação para organização e posterior commit, podemos iniciar o commit com o comando git commit, no Git Bash. Com o git log, podemos ver algumas informações, como o autor e e-mail, o qual definimos anteriormente, a branch master, a data do commit e o hash do commit, basicamente a sua identidade (o hash do commit está logo abaixo do comando git log, é a sequência de números e letras).

Em nossos commits as vezes tem pastas em que não queremos que apareçam em nossos commits, para isso, podemos criar um bloco de notas chamado .gitignore, com o nome da pasta dentro do arquivo. Para fazermos isso, basta digitar o comando echo pasta/ > .gitignore, com esse comando, o bloco de notas será criado e a pasta indicada será ignorada pelo Git. Há também um arquivo com a função oposto ao .gitignore, o .gikeep, que serve para salvar as pastas que estão sem arquivos, pois quando uma pasta não possui arquivos dentro de si, ela não é considerada no git add.

Enviando e Baixando Alterações com o Repositório Remoto

Com os nossos arquivos locais prontos para upload, podemos enviar-los para o repositório remoto no GitHub, para isso, precisaremos criar uma conexão entre o nosso repositório local com o repositório remoto.

Para fazermos isso, começaremos com o comando git branch -M main, para forçar que a branch principal seja renomeada para main; git remote add origin "URL", para criar a conexão entre os respositórios; git push -u origin main, para enviarmos os arquivos do repostório local para o repositório remoto.

Para realizarmos o contrário, ou seja, fazer o download de arquivos do repositório remoto, precisaremos utilizar o comando git pull, em nosso Git Bash, após todo o processo de conexão entre os repositórios.
