Configurando o Git

Para iniciarmos a nossa jornada com a criação de grandes repositórios que irão conter os nossos projetos, precisamos primeiro configurar devidamente o nosso DVCS, para que não haja problemas de autenticidade mais para frente.

Com isso, iniciaremos com o download do Git Bash, o DVCS do Git.

Agora, criamos uma nova pasta para iniciar a nossa jornada com o Git Bash.

Com o Git Bash aberto, nos deparamos com essa tela:

Nesta tela inicial, podemos ver três coisas importantes:

  1. Nome de Usuário
  2. Nome do Computador
  3. Endereço da pasta em que abrimos o Git Bash
Em outras palavras, usuário@nome do computador, nome do computador ~/endereço da pasta.

Para sabermos a variedade de coisas que podemos fazer, basta digitar git config

Quando nos deparamos com a lista de comandos, é importante saber o que os três primeiros representam, --global, --system e --local, que são os "lugares" em que as variáveis ficam armazenadas.

  1. --global: Referencia todo tipo de alteração feita no usuário
  2. --system: Referencia todo o tipo de alteração feita ao sistema como um todo, como exemplo, o Windows
  3. --local: Referencia todo o tipo de alteração feita em um repositório específico

Agora, podemos configurar o nosso nome e e-mail do git, com os comandos git config --global user.name e git config --global user-email, que ficará assim:

Para verificar a lista de configurações realizadas, utilizamos o comando git config --global --list, sendo --global porquê é referente as configurações realizadas ao usuário.
