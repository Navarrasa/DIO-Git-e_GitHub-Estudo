Version Control System

Sistema de Controle de Versão (VCS), são utilizados na Engenharia de Software para gerenciar as alterações em programas, projetos, documentos e outras coleções de informações. Este sistema facilita na organização de dados, na agilidade e visualização geral dos dados. Um exemplo de agilidade que o VCS nos traz: Um programador termina o seu código em um novo aplicativo para dispositivos Android, porém, um bug, um erro inesperado aconteceu. Utilizando o VCS, o programador pode baixar um código antes das modificações realizadas e comparar para encontrar o motivo do bug e corrigir-lo mais rapidamente.

Local Version Control System

O Sistema de Controle de Versão Local é aquele em que os arquivos estão localizados no próprio computador em uma pasta onde se cria diferentes versões para o mesmo arquivo, como exemplo arquivo.1, arquivo.2, e por assim em diante. Não é muito recomendado utilizar este método de Versionamento de Código (VCS), pois todo o conteúdo estará em um único arquivo, em um único computador, ou seja, outros programadores não conseguirão trabalhar com o conteúdo do diretório, e caso haja algum problema com o computador em que estão armazenados, tudo será perdido.

A ferramenta mais conhecida para VCS Locais é o RCS (Revision Control System), que funciona mantendo conjuntos de alterações (ou seja, as diferenças entre os arquivos) em um formato especial no disco; ele pode, em seguida, re-criar como qualquer arquivo se parecia em qualquer ponto no tempo, adicionando-se todas as alterações.

Centralized Version Control System

O Sistema de Controle de Versão Centralizado é aquele em que há um servidor que contém todos os arquivos e um grande números de desenvolvedores que podem acessar este servidor, e atuar no mesmo projeto. Diferente do VCS Local, o VCS Centralizado é mais eficiente pois agora você sabe onde o seu colega está trabalhando no código, o gerenciamento dos arquivos por parte do administrador é muito mais ágil, pois todos os arquivos trabalhados estão em um único servidor. Mesmo sendo um VCS muito utilizado por trazer agilidade, havia ineficiências que foram percebidas ao longo do tempo. A principal ineficiência era a extrema dependência do servidor para a atualização dos arquivos trabalhados, ou seja, caso o servidor ficasse fora do ar, insidponível, toda a equipe seria prejudicada, e caso houvesse alguma falha durante o processo de gravação de arquivos no servidor, todo o trabalho do desenvolvedor seria perdido.

Alguns sistemas muito conhecidos que foram utilizados no VCS Centralizado foram o CVS, Subversion e Perforce.

Distributed Version Control System

O Sistema de Controle de Versão Distribuído é aquele em que há o servidor com os arquivos, porém os desenvolvedores conseguem clonar os arquivos para um repositório local, assim, caso o servidor fique indisponível, o prórpio repositório do desenvolvedor pode ser utilizado para restaurar o servidor. Cada clone é um backup de todos os dados do servidor. Além disso muitos sistemas trabalham com diversos repositórios, ou seja, é possível colaborar em diferentes grupos de pessoas ao mesmo tempo dentro do mesmo projeto, pois todos os arquivos serão iguais e todos tem acesso a eles.

Algumas ferramentas muito utilizadas para DVCSs, são: Git e Mercurial, as duas ferramentas mais utilizadas globalmente por terem muitos recursos e serem simples de manusear. o Git é muito utilizado em projetos open source e possui características marcantes como a sua velocidade, que é a maior que os demais sistemas por necessitar apenas de recursos locais. O Mercurial é utilizado em desenvolvimento de projetos de grande porte, apresentando melhor resultado em SO's (Sistemas Operacionais), compatíveis com a linguagem de programação Python, que é a linguagem em que o Mercurial foi desenvolvido.
