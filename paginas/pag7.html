Criando Branches

As Branchs, ou "ramos", são ramificações de seu projeto, ponteiros que demarcam os commits realizados, sendo trabalhadas de maneira independente, isso porquê podemos criar diversas branchs durante os processos no Git, para que possamos adicionar funcionalidades em nosso projeto, sem alterar a branch principal, em que está sendo desenvolvida.

Para exemplificar, veja a imagem abaixo:

Em nossos commits, podemos ver o branch em que se está sendo utilizada. Para visualizar os commits realizados, digitamos git log, onde mostrará todos os commits e em qual branch eles estão sendo desenvolvidos.


Observe que: No log, podemos observar o nosso user.name e user.email, previamente configurados.
Ao lado do ID do commit, está a branch utilizada.

Para criar uma nova branch, usamos o seguinte comando no Git Bash: git chekout -b nomedabranch
Este comando criará uma nova branch e essa branch estará sendo desenvolvida a partir do último commit realizado pela branch principal (main ou master)

Para alterarmos novamente para a branch principal, use o comando git chekout main/master (O motivo de não ter o -b desta vez é porquê estamos trocando de branch e não criando uma nova, fica a dica!)

Se quisermos ver a lista de branchs disponíveis e seus últimos commits, podemos utilizar o comando git branch -v

Mesclando Branches

Caso seja necessário que as alterações da branch nova seja incluída à branch principal, precisaremos mesclar elas.
Nesse caso, para mesclar uma branch, usaremos o comando git merge nomedabranch, e essa branch selecionada será mesclada à branch principal

Deletando Branches

Quando a branch deixar de ser utilizada durante o desenvolvimento de uma nova função, podemos deletar ela pelo Git Bash com o comando git branch -d nomedabranch.

Trantando de Conflitos e Comandos úteis

Conflitos de Merge

Ocorre quando há alterações concorrentes entre dois colegas de equipe, acarretando em um arquivo com as duas versões trabalhadas. Para tratarmos deste conflito, os colegas deverão escolher qual das versões eles irão manter, ou desfazer as alterações, para que o problema seja cessado

Comandos úteis do dia a dia

Baixando conteúdo da Branch remota sem mesclar com a Branch local

Usamos o comando git fetch origin/main sendo origin referente ao repositório remoto e main referenciando a branch principal. Após isso, podemos utilizar o comando git merge, para baixarmos o conteúdo da branch.

Selecionando uma branch para clonar

Há vezes em que nos deparamos com repositórios que estão cheios de diferentes branchs e precisamos apenas de uma para fazer o download. É bem simples selecionar a branch, para isso, abrimos o Git Bash, e realizamos o comando git clone URL --branch nomedabranch --single-branch. Caso não seja especificado o nome da branch solicitada, o comando irá clonar apenas a branch principal.

Armazenar alterações

Quando precisamos alterar branchs, mas há modificações pendentes e não queremos que essa modificação seja aplicada na branch, usamos o comando git stash, para armazenar as alterações realizadas. Para visualizar a quantidade de stashs, temos o comando git stach list. Com essas alterações armazenadas, temos duas opções de comando, git stash pop, para trazer as alterações armazenadas e excluir as alterações recentes ou git stash apply, para manter as modificações armazenadas.
